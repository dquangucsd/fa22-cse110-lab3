Food App UI Planning

We discuss how far the team is on how they want to implement the food app.

Navi Bar:

Alejandro is considering putting the navi bar at the top or bottom. Can we fit other things?
Bar at the left or right is ruled out, we can implement an option to move it later.

Why no left or right?

Phone screens are tall not wide, it would make the ratio even worse.
We would also then need the user to choose a side to put the navi bar based on whether they're left or right handed.

Tom and David are creating a prototype version with an adaptive bar based on what page the user is on.

  1. Consider what are essentials for each page. Don't forget back button
  2. Decide which features to go on, minimize or remove unnecessary ones
  3. Communicate with settings team to allow user to manually add more features
    If you need to communicate: dquang@ucsd.edu

Order Page:

Clarence and Winnie are designing two versions, a list and scrolling cards.

  • List Pros

    • Simple
    • Can use green dot for ready, yellow for in progress, red for not received

  • List Cons

    • Can overload user with text if not managed well
    • Not as many animations so not as cool

  • Scrolling Pros

    • Looks cool
    • Bigger text, doesn't overload user with text, more animations

  • Scrolling Cons

    • Harder if many separate orders
    • Can be more cumbersome
